\begin{abstract}
This report documents work toward inferring a Variable-Structure Automaton (VSA)
from a unique dataset of state--action--feedback tuples generated in a P-model
environment. We summarize the mathematical background for empirical counts and
conditional estimates, the methodology for Tasks~1--4
(estimation, optimal-action identification, $\varepsilon$-optimality, and
simulation) in mathematical terms, and outline planned work for
Tasks~5--6. Implementation scripts live under \texttt{../proj\_python/}; excerpts
appear in Appendix~\ref{app:code}.
\end{abstract}

\section{Introduction}

The term project asks us to reconstruct, from observed sequences only, the
state--action policy, transition dynamics, penalty structure, and norm of
behavior of an unknown automaton with five states
$\{\phi_1,\ldots,\phi_5\}$, three actions $\{\alpha_1,\alpha_2,\alpha_3\}$,
and binary environment feedback $\beta\in\{0,1\}$ (P-model). Each dataset row is
a tuple
$(\text{sequence\_id},\,\text{step},\,\Phi_{\mathrm{src}}(n),\,\Phi_{\mathrm{dst}}(n),\,\alpha(n),\,\beta(n))$.
This document aligns notation with the course assignment and with the
implementation under \texttt{../proj\_python/}.

\section{Background}\label{sec:background}

\subsection[Empirical count N(phi, alpha)]{Empirical count $N(\phi_i,\alpha_k)$}

The number of times action $\alpha_k$ was taken while the source state was
$\phi_i$ is
\begin{equation}
  N(\phi_i,\alpha_k)
  = \#\bigl\{\, n : \Phi_{\mathrm{src}}(n)=\phi_i \;\wedge\; \alpha(n)=\alpha_k \bigr\}.
\end{equation}
In the CSV, $\Phi_{\mathrm{src}}(n)$ is the column \texttt{phi\_src} and
$\alpha(n)$ is \texttt{alpha}.

\subsection[Policy estimate]{Policy estimate $P(\alpha_k\mid\phi_i)$}

Empirical policy probabilities normalize the counts in each state:
\begin{equation}
  P(\alpha_k\mid\phi_i)
  = \frac{N(\phi_i,\alpha_k)}{\sum_{k'} N(\phi_i,\alpha_{k'})}
  = \frac{\text{times }\alpha_k\text{ taken in }\phi_i}
         {\text{times }\phi_i\text{ was visited}}.
\end{equation}
The denominator is the total number of actions chosen from state $\phi_i$.

\subsection{Row totals \texttt{row\_totals\_phi}}

In code, for each state $\phi_i$,
\begin{equation}
  \texttt{row\_totals\_phi}[\phi_i] = \sum_k N(\phi_i,\alpha_k),
\end{equation}
i.e.\ the sum of the $N(\phi_i,\cdot)$ row in the count matrix \texttt{N\_phi\_alpha}.

\subsection{Transition and penalty counts (brief)}

Similarly, $N(\phi_i,\alpha_k,\phi_j)$ counts transitions
$\phi_i\to\phi_j$ under action $\alpha_k$, and $N(\alpha_k,\beta)$ counts
(action, feedback) pairs. These yield transition probabilities
$P(\phi_j\mid\phi_i,\alpha_k)$ and penalty probabilities
$c_k = P(\beta=1\mid\alpha_k)$ by row-wise normalization of the appropriate
count tables.

\subsection{Task 1 --- Estimation}

We load the headerless UTF-8 CSV, build \texttt{N\_phi\_alpha} by grouping on
\texttt{phi\_src} and \texttt{alpha}, build per-action transition count
matrices \texttt{N\_phi\_alpha\_phi}, build \texttt{N\_alpha\_beta}, then
derive \texttt{policy}, \texttt{transition}, and \texttt{penalty} ($c_k$).
Let $r_i=\sum_k N(\phi_i,\alpha_k)$. For visited states ($r_i>0$),
$P(\alpha_k\mid\phi_i)=N(\phi_i,\alpha_k)/r_i$; for each action $\alpha_k$,
transition probabilities normalize each row of $N(\phi_i,\alpha_k,\phi_j)$ over
$\phi_j$. Penalties use
$c_k = N(\alpha_k,1)\big/\sum_{\beta\in\{0,1\}} N(\alpha_k,\beta)$,
equivalently the empirical mean of $\beta$ conditional on $\alpha_k$.
These operations realize the count-based policy in Section~\ref{sec:background}
and extend the same row-normalization idea to transition tables and to
$c_k$ from $N(\alpha_k,\beta)$.
\noindent The companion code is in \texttt{estimation.py} (entry point \texttt{run\_estimation});
shared types and column constants live in \texttt{util.py}.
Appendix~\ref{app:estimation} reproduces lines~117--164 (counts through
\texttt{policy}, \texttt{transition}, \texttt{penalty}, and basic consistency checks).

\subsection{Task 2 --- Optimal action identification}

Under the P-model, the optimal action minimizes the penalty probability:
\begin{equation}
  \alpha^\ast \in \arg\min_k c_k,
  \qquad c_k = P(\beta=1\mid\alpha_k).
\end{equation}
The implementation reads $c_k$ from the in-memory \texttt{EstimationResult}
when called from \texttt{main.py}, or from \texttt{penalty.csv} after a
standalone Task~1 run. Ties at the minimum are resolved deterministically
(first label in sorted order). Computationally,
$\alpha^\ast = \arg\min_k c_k$ among the empirical $c_k$ from Task~1.
\noindent The companion code is in \texttt{optimal\_action\_identification.py}
(entry point \texttt{run\_optimal\_action}); Appendix~\ref{app:optimal} shows lines~110--131
(tie-breaking at the minimum $c_k$ and construction of the result object).

\subsection[Task 3 --- epsilon-optimality]{Task 3 --- $\varepsilon$-optimality}

Given $\alpha^\ast$ from Task~2 and the policy $P(\alpha\mid\phi)$ from Task~1,
the assignment defines
\begin{equation}
  \varepsilon = 1 - \min_i P(\alpha^\ast\mid\phi_i),
\end{equation}
where the minimum is taken over \textbf{visited} states $\phi_i$, i.e.\ rows of
\texttt{N\_phi\_alpha} with positive row sum (states that appear in the data).
States never visited are excluded so the minimum reflects empirical support.
Let $\mathcal{V}=\{\phi_i : \sum_k N(\phi_i,\alpha_k)>0\}$; then equivalently
$\varepsilon = 1 - \min_{\phi_i \in \mathcal{V}} P(\alpha^\ast\mid\phi_i)$.
The same quantity can be evaluated from stored matrices or from Task~1--2
outputs on disk.
\noindent The companion code is in \texttt{e\_optimality\_analysis.py}; helper
\texttt{\_compute\_core} evaluates the minimum over visited states, and
\texttt{run\_epsilon\_optimality} dispatches in-memory \texttt{EstimationResult} /
\texttt{OptimalActionResult} \emph{vs.}\ CSV inputs from Tasks~1--2.
Appendix~\ref{app:epsilon} lists lines~110--130.

\subsection{Task 4 --- Simulation}

Synthetic trajectories use the learned \textbf{policy}, \textbf{transition}
matrices, and \textbf{penalty} vector $c_k = P(\beta{=}1\mid\alpha_k)$. For each
real sequence ID we keep the \textbf{same length} and \textbf{same starting}
$\Phi_{\mathrm{src}}$ as in the dataset; each step samples
$\alpha \sim P(\cdot\mid\phi)$, then $\phi' \sim P(\cdot\mid\phi,\alpha)$, then
$\beta \sim \mathrm{Bernoulli}(c_\alpha)$. Marginals of actions, $\phi_{\mathrm{src}}$,
$\phi_{\mathrm{dst}}$, and penalty rates are compared to the real CSV; the code
reports $|P_{\mathrm{data}}-P_{\mathrm{sim}}|$ per category (and L1-style sums).
If a transition row is missing for the current $\phi$, a uniform next-state
distribution is used so sampling remains well-defined.
Marginal distributions are empirical frequencies over rows (actions,
$\Phi_{\mathrm{src}}$, $\Phi_{\mathrm{dst}}$); penalty comparison uses per-action
sample means of $\beta$ and the global mean.
\noindent The companion code is in \texttt{simulation.py}; \texttt{\_simulate\_block}
implements one synthetic dataset aligned with real sequence lengths and initial
states, and \texttt{run\_simulation} loads the real CSV, compares marginals, and
aggregates absolute differences (from line~246 onward in the source file).
Appendix~\ref{app:simulation} shows lines~134--184.

\section{Conclusion}

We fixed notation for empirical counts and policy normalization, and linked
Task~1 estimates to the count-based constructions in Section~\ref{sec:background}.
Task~2 selects
$\alpha^\ast$ by minimizing $c_k$; Task~3 quantifies $\varepsilon$-optimality
via $\min_i P(\alpha^\ast\mid\phi_i)$ over visited states; Task~4 validates the
generative model against data marginals. Remaining work covers perturbation
robustness and critique of the P-model assumption.
